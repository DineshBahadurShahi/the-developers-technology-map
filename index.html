Introduction
Technology Ecosystem Map

Every technology in this book fits one of these layers. Start here to see the whole picture before diving into any chapter.

How to read this map

Each layer depends on the one below it. Frontend sits on top of backend. Backend uses databases and the AI layer. Everything runs on cloud infrastructure. Real-time systems cut across all layers.

Part 1 · The Big Picture
How Software Actually Works

The mental model every developer needs before learning any specific technology.

The three-tier model

Almost every piece of software follows the same pattern: something the user sees (frontend), something that processes requests (backend), and somewhere data lives (database). Instagram, Stripe, Notion, GitHub — all three tiers.

Frontend
What the user sees. Runs in the browser. HTML, CSS, JavaScript.
Backend
Processes logic and requests. Enforces rules. Node.js, APIs.
Database
Stores all persistent data. The source of truth. Postgres, Redis.

What happens when you click "Send" on Slack

1
Browser captures your input
JavaScript reads the text field and fires the click event.
2
HTTPS request leaves your device
An encrypted POST request carries your message to Slack's servers.
3
DNS resolves the address
Your device asks "where is slack.com?" and gets back an IP address.
4
Load balancer routes the request
Slack's infrastructure picks which backend server handles this request.
5
Backend saves to database
The message is validated, stored in Postgres, and confirmation sent.
6
WebSocket pushes to recipients
Everyone in the channel receives the message instantly.
Key insight

Every technology in this book plays one of these roles. When you learn something new, always ask: "Which step does this handle?"

Part 1 · The Big Picture
The Journey of a Request

Every time you open a website, dozens of technologies collaborate in milliseconds.

What this tells you

Modern software is not a single program — it is a network of collaborating services. Each technology in this book handles one piece of this puzzle. Understanding where each piece fits is the purpose of this entire book.

The city metaphor

DNS is the address system. CDN is the post office. Backend servers are the offices doing actual work. The database is the filing room. Load balancers are the traffic directors. Each technology has a job.

Part 2 · Frontend
HTML

The skeleton of every webpage. Not a programming language — a structure language.

What is it?

HTML (HyperText Markup Language) describes the structure and content of a webpage. Think of it as a building's blueprint — it defines what's there and in what order, not how it looks.

Why does it exist?

In 1989, Tim Berners-Lee needed a way to share scientific documents over the internet. He invented HTML so browsers could distinguish a heading from a paragraph from a link. Every other web technology is built on top of that original idea.

Real example

Every YouTube video page, Wikipedia article, and GitHub profile is HTML structuring the content. The words, images, buttons, and sections are all HTML elements.

Where it fits

HTML lives entirely in the browser. It is the structure layer. CSS styles it. JavaScript makes it interactive. React generates it dynamically. Server-side frameworks render it on the server first.

Technology connections

CSS → styles HTML
JavaScript → manipulates HTML
React → generates HTML
Next.js → server-renders HTML

Common beginner mistakes

MistakeReality
"HTML is a programming language"It's markup — structure, not logic.
Using <div> for everythingSemantic tags like <nav>, <main>, <article> help SEO and accessibility.
Not closing tagsUnclosed tags cause unexpected layout bugs.
One-Minute Summary

HTML is the structure layer of every webpage. It defines what's on the page. Every frontend technology — CSS, JavaScript, React — builds on top of it. Learn it first. The basics take 2–3 days. You'll use it every day.

Part 2 · Frontend
CSS

CSS makes things look good. It turns a bare HTML skeleton into a beautiful interface.

What is it?

CSS (Cascading Style Sheets) controls how HTML looks: colors, fonts, layouts, spacing, animations. Without CSS, every webpage would look like a plain 1992 text document.

Why does it exist?

Originally styling was done inline in HTML attributes. CSS was invented to separate content from presentation. Change one CSS file and the look of every page changes instantly.

Real example

Apple's website uses a single CSS change to switch between light and dark mode. Linear, Vercel, and Stripe — their distinctive identities are entirely CSS on top of basic HTML.

Flexbox
One-dimensional layout. Rows or columns. Used everywhere.
Grid
Two-dimensional. Rows AND columns simultaneously.
Responsive
Media queries make designs adapt to screen sizes.
Custom Props
CSS variables for reusable values across your stylesheet.

Technology connections

HTML → CSS styles it
Tailwind → utility-first CSS
React → CSS Modules / styled-components
One-Minute Summary

CSS makes HTML beautiful. Learn Flexbox and Grid first — they unlock 90% of layouts. Without CSS you have content. With CSS you have design.

Part 2 · Frontend
JavaScript

The only programming language that runs natively in the browser. The backbone of modern web development.

What is it?

JavaScript is a programming language that makes webpages interactive. It adds behavior — clicking, animations, fetching data, validating forms. Today it also runs on servers (Node.js), in mobile apps, and in AI tooling.

Why does it exist?

In 1995, Netscape needed form validation without a full page reload. Brendan Eich created JavaScript in 10 days. It became the only language browsers execute natively — and then the language of the entire software world.

Real example

Google search results appearing as you type — JavaScript. Twitter's like button turning red without a page reload — JavaScript. Dragging a Trello card between columns — JavaScript.

Key concepts every developer must know

ConceptWhat it doesPriority
async/awaitHandle operations that take timeCritical
DOM manipulationChange the page dynamicallyCritical
Array methodsmap, filter, reduceCritical
ClosuresFunctions that remember their scopeImportant
PrototypesJavaScript's inheritance modelImportant
One-Minute Summary

JavaScript is the language of the web. HTML is structure. CSS is style. JavaScript is behavior. It's the most-used language on Earth. Learn it deeply — everything else in frontend and full-stack builds on it.

Part 2 · Frontend
TypeScript

JavaScript with a safety net. TypeScript catches your mistakes before they reach production.

What is it?

TypeScript is a superset of JavaScript that adds optional static types. Every JavaScript file is valid TypeScript. But TypeScript lets you declare that a variable is a number or a function returns a string — and it warns you when something doesn't match.

Real example

Vercel, Stripe, Airbnb, and Microsoft all use TypeScript in production. In a 500-file codebase, TypeScript prevents entire categories of bugs that tests alone can't catch.

Autocomplete
Your editor knows every property on every object.
Refactoring
Rename a function and TypeScript updates every call site.
Self-documenting
Types tell the next developer exactly what a function expects.
Fewer bugs
Wrong type used? Caught at dev time, not in production.
One-Minute Summary

TypeScript is JavaScript with types. Learn JavaScript first, then add TypeScript. It makes your code safer, your editor smarter, your team more productive. In 2025, most professional projects use TypeScript by default.

Part 2 · Frontend
React

The most popular way to build user interfaces. Created by Meta. Used by almost every major tech company.

What is it?

React is a JavaScript library for building user interfaces through components. Instead of manually updating the DOM, you describe what the UI should look like for any given state — and React handles all the updates efficiently.

The core idea: components

React breaks your UI into components — small, reusable pieces. A Button component. A Card component. A Navigation bar. Each manages its own state and renders its own HTML. You compose them like LEGO blocks.

Real example

Facebook, Instagram, WhatsApp Web, Airbnb, Netflix — all use React. When you click "Like" and the count changes without a page reload, React is managing that state update.

ConceptWhat it means
ComponentsFunctions that return JSX (HTML-like syntax)
PropsData passed into a component from its parent
State (useState)Data that belongs to a component and can change
Effects (useEffect)Fetch data, subscriptions, timers
Virtual DOMReact's mechanism for efficient UI updates
Common beginner mistake

Jumping into React before understanding JavaScript. React is JavaScript — if you don't know functions, arrays, and async/await, React will feel like magic. Spend a month on JS first.

One-Minute Summary

React turns UI building from "find the element and change it" into "describe what it should look like." It's the most in-demand frontend skill. Learn JavaScript → React → Next.js in that order.

Part 2 · Frontend
Next.js

The React framework for production. Routing, server rendering, API routes, and optimization — built in.

What is it?

Next.js is a full-stack framework built on React. React gives you components. Next.js adds everything a real application needs: file-based routing, server-side rendering, API endpoints, image optimization, and performance defaults.

App Router
A file at /app/about/page.tsx becomes yoursite.com/about automatically.
Server Components
Components that render on the server. Zero JS sent to the client.
API Routes
Backend endpoints inside your app. No separate server needed.
Edge Runtime
Code that runs at CDN edge, globally, near your users.
Real example

The Vercel website, OpenAI's ChatGPT interface, TikTok's web app — all run on Next.js. It's the default choice for serious React applications in 2025.

One-Minute Summary

Next.js = React + routing + server rendering + API routes + optimization. It turns React from a UI library into a complete web framework. The most popular full-stack JavaScript framework today.

Part 2 · Frontend
Tailwind CSS

Write CSS without writing CSS. Utility classes directly in your HTML or JSX.

What is it?

Tailwind is a utility-first CSS framework. Instead of a class called .card with CSS properties, you apply small single-purpose classes directly: bg-white p-4 rounded-lg flex items-center gap-3. The HTML shows you exactly what it looks like.

No naming
Never agonize over .card-inner-wrapper again.
No switching files
Styles inline next to your HTML. One place to look.
Consistent scales
Pre-defined spacing, colors, and typography.
Tiny output
Purges unused CSS. Ships only what you use.
Real example

Vercel docs, shadcn/ui, and thousands of SaaS products are built with Tailwind. It pairs perfectly with React and Next.js — rarely seen without the others in 2025 codebases.

One-Minute Summary

Tailwind replaces CSS files with utility classes. Learn CSS fundamentals first — then Tailwind makes you much faster. The modern default stack: Next.js + TypeScript + Tailwind.

Part 3 · Backend
Node.js

JavaScript escaped from the browser. Node.js runs JavaScript on servers — changing the web forever.

What is it?

Node.js is a JavaScript runtime for servers. Before Node.js, JavaScript only ran in browsers. Node.js took Chrome's V8 engine and made it run on computers, servers, and in the cloud.

Real example

Netflix moved to Node.js and cut startup time by 70%. LinkedIn reduced servers from 30 to 3. Uber, PayPal, and NASA all use Node.js in their backend.

Use caseExample
REST APIsExpress.js, Fastify, Hono
Real-time appsWebSocket servers, chat
CLI toolsnpm, Vite, ESLint are all Node.js
Serverless functionsVercel Functions, AWS Lambda
One-Minute Summary

Node.js = JavaScript on the server. It's what Next.js API routes run on. It's what npm is built on. If you know JavaScript, you already know how to write backend code with Node.js.

Part 3 · Backend
APIs & REST

The language services use to talk to each other. Every modern application is built on APIs.

What is an API?

An API (Application Programming Interface) is a contract between two pieces of software. "Send me a request in this format, and I'll send you data in that format." Stripe's API lets you charge a card. OpenAI's API gives you an AI model.

What is REST?

REST is the most common style for building APIs. It uses HTTP methods to perform operations on resources: GET /users/42 means "get me user 42." DELETE /posts/7 means "delete post 7."

MethodActionExample
GETReadGET /products
POSTCreatePOST /orders
PUTUpdatePUT /users/5
DELETEDeleteDELETE /posts/3
Real example

When you pay with Stripe, your app calls Stripe's REST API. When you sign in with Google, Google's OAuth API is called. You interact with REST APIs hundreds of times per day.

One-Minute Summary

APIs are how software talks to software. REST is the most common pattern. You'll call APIs from your frontend and build them in your backend. Understanding REST — methods, status codes, JSON — is non-negotiable.

Part 3 · Backend
GraphQL

Ask for exactly the data you need. Nothing more. Nothing less.

What is it?

GraphQL is a query language for APIs created by Meta. Instead of hitting multiple REST endpoints and receiving data you didn't ask for, you send one query specifying exactly which fields you need — and get back precisely that.

RESTGraphQL
EndpointsOne per resourceSingle /graphql
Data returnedFixed shape, often excessExactly what you asked for
Multiple resourcesMultiple requestsOne query, nested
Best forSimple CRUD, public APIsComplex data, mobile apps
When you actually need it

GraphQL adds complexity. For most apps REST is simpler and sufficient. Choose GraphQL when you have complex nested data, multiple client types needing different shapes, or a strongly-typed API contract requirement.

Part 3 · Backend
Authentication & OAuth

Proving identity is one of the hardest problems in software. Here's how the industry solves it.

Authentication vs Authorization

Authentication
"Who are you?" Verifying identity. Login with email/password or Google.
Authorization
"What can you do?" Checking permissions. Admins vs regular users.

How OAuth works

1
User clicks "Sign in with Google"
Your app redirects to Google with your app's client ID.
2
Google authenticates the user
Google handles the login. Your app never sees the password.
3
Google returns an auth code
Redirects back to your app with a one-time code.
4
Your server exchanges it for tokens
Access token (short-lived) and refresh token (long-lived).
5
User is logged in
You store identity. They stay logged in via JWT or session.
Auth.js

Auth.js (formerly NextAuth) is the most popular authentication library for Next.js. Add 20 lines of config and you get sign-in with Google, GitHub, Discord, and email magic links — all securely implemented.

Part 3 · Backend
Serverless

There are still servers. You just don't manage them. Serverless means "server-ful but invisible."

What is it?

Serverless means writing individual functions that run on demand. The cloud provider handles provisioning, scaling, and maintenance. You pay per invocation, not for a server running 24/7.

Traditional Server
Runs 24/7. You pay constantly. You manage updates and scaling yourself.
Serverless Function
Runs on demand. Scales automatically. Pay per call. Zero ops.
Real example

Every Next.js API route you write is a serverless function on Vercel. Stripe webhooks are serverless functions. An AI image endpoint that runs 10 seconds occasionally — perfectly suited to serverless.

When to avoid serverless

Long-running processes over 15 minutes, WebSocket connections, and anything needing persistent in-memory state don't suit serverless. Use a traditional server or container for those cases.

Part 4 · Databases
SQL

The language of relational databases. 50 years old and still dominant.

What is it?

SQL (Structured Query Language) is used to interact with relational databases. Data is stored in tables — rows and columns like a spreadsheet. Tables link to each other via foreign keys. You query with SQL statements.

CommandActionExample
SELECTRead dataSELECT * FROM users WHERE age > 18
INSERTAdd dataINSERT INTO posts (title) VALUES ('Hello')
UPDATEModifyUPDATE users SET name = 'Din' WHERE id = 1
DELETERemoveDELETE FROM sessions WHERE expired = true
JOINCombine tablesSELECT * FROM orders JOIN users ON user_id = users.id
Why developers love it

SQL is declarative — you say what you want, not how to get it. Learn SQL once and use it on PostgreSQL, MySQL, SQLite, and more. It's been battle-tested for 50 years.

Part 4 · Databases
PostgreSQL

The world's most advanced open-source relational database. The default for serious applications.

What is it?

PostgreSQL (Postgres) is an open-source relational database with 35+ years of development. Known for reliability, correctness, and features that go far beyond standard SQL.

JSON support
Store and query JSON natively. Best of relational and document worlds.
Extensions
pgvector for AI. PostGIS for geo. TimescaleDB for time-series.
ACID compliant
Transactions are safe. Data is never corrupted. Banks use Postgres.
Open source
No licensing cost. Backed by a huge community. Runs everywhere.
Real example

Instagram, Reddit, Twitch, Discord, and GitHub all use PostgreSQL. Supabase is "Firebase but Postgres." Neon, PlanetScale, Railway, and Render all offer managed Postgres. The overwhelmingly dominant choice for startups in 2025.

The Prisma connection

Most modern apps use Prisma as an ORM on top of Postgres. Define your schema in a .prisma file. Prisma generates type-safe queries in TypeScript. No raw SQL needed for most operations.

Part 4 · Databases
MongoDB

A document database. No tables, no rigid schemas — store JSON objects directly.

What is it?

MongoDB is a NoSQL document database. Instead of rows in tables, you store documents — JSON-like objects. A user document might have a name, email, an array of addresses, and nested preferences — all in one document, no joins required.

SituationChoose
Structured data with relationshipsPostgreSQL
Flexible schema that changes oftenMongoDB
E-commerce, banking, SaaSPostgreSQL
Content management, catalogsMongoDB
Need ACID transactionsPostgreSQL
Honest take

MongoDB was extremely popular in the 2010s (MEAN stack). Today Postgres with JSON support and Prisma handles most MongoDB use cases — with the added benefit of transactions and relational integrity. Default to Postgres for new projects.

Part 4 · Databases
Redis

The fastest data store on Earth. In-memory, blazing fast, used for caching, sessions, and real-time features.

What is it?

Redis is an in-memory data store. All data lives in RAM, so reads and writes happen in microseconds. Used for caching expensive queries, storing user sessions, rate limiting, and pub/sub messaging.

→
User requests dashboard data
Check Redis first: is this data in cache?
✓
Cache hit → return in under 1ms
User gets data instantly. No database query needed.
✗
Cache miss → query Postgres
Get data from database, store in Redis for next time.
Caching
Cache expensive DB queries. Reduce load. Speed up responses dramatically.
Sessions
Store user sessions. Fast auth checks on every request.
Rate limiting
Limit API calls per user per minute. Prevent abuse.
Pub/Sub
Real-time messaging between services.
Part 5 · Cloud & Infrastructure
DNS & Domains

The internet's address book. DNS translates human-readable names into machine addresses.

What is DNS?

DNS (Domain Name System) maps domain names (google.com) to IP addresses (142.250.80.46). Computers communicate via IPs. Humans use domain names. DNS bridges the gap — in milliseconds, globally, billions of times per second.

DNS record types every developer needs

RecordPurposeExample
AMaps domain to IPv4myapp.com → 104.21.33.92
CNAMEAlias to another domainwww → myapp.com
MXMail server routingGoogle Workspace email setup
TXTVerification, SPF, DKIMProve domain ownership
Practical tip

When you deploy to Vercel and add a custom domain, you're adding a CNAME record in your domain registrar (Namecheap, Cloudflare) pointing to Vercel's DNS. That's DNS in practice — you'll do it on every project.

Part 5 · Cloud & Infrastructure
Vercel & Modern Hosting

Deploy a production application in 30 seconds. Modern hosting abstracts away what used to require a DevOps team.

What is Vercel?

Vercel is a cloud platform optimized for frontend and full-stack JavaScript, built by the creators of Next.js. Connect a GitHub repo, and every push auto-deploys. Every PR gets a preview URL. Serverless functions are automatic. Global CDN included.

PlatformBest forComplexity

For any Next.js project: Vercel for the app, Supabase (managed Postgres) for the database, Upstash (managed Redis) for caching. Idea to production in hours, not weeks.

Part 5 · Cloud & Infrastructure
AWS

The cloud infrastructure powering a third of the internet. Essential to understand, careful when to start with.

What is it?

AWS (Amazon Web Services) is the world's largest cloud computing platform with 200+ services. Netflix runs almost entirely on AWS. Airbnb, Slack, Reddit, and NASA all use AWS.

EC2
Virtual servers. The basic compute unit of AWS.
RDS
Managed PostgreSQL, MySQL. No server management.
S3
Object storage. Images, files, backups — unlimited scale.
Lambda
Serverless functions. Run code without managing servers.
CloudFront
AWS's CDN. Serve content fast globally.
IAM
Control who can do what on your AWS account.
Honest advice

Don't start with AWS for side projects. Start with Vercel + Supabase. Learn AWS when you join a company that uses it, or when you need enterprise-scale control. AWS is powerful but complex — and expensive if misconfigured.

Part 5 · Cloud & Infrastructure
Docker

"It works on my machine." Docker solves this by packaging your app and its entire environment together.

What is it?

Docker is a platform for packaging applications into containers. A container includes your code, runtime, dependencies, and configuration. It runs identically on your laptop, a colleague's computer, a CI server, and a cloud VM.

Dockerfile
A recipe describing how to build your container image.
Image
The built, read-only template from a Dockerfile.
Container
A running instance of an image. Lightweight, isolated, disposable.
Compose
Run App + Postgres + Redis together with one command.
Real example

New developer joining. Without Docker: 3 hours installing and configuring dependencies. With Docker: docker-compose up and everything runs in 3 minutes. Same versions. Same behavior. Every time.

Part 5 · Cloud & Infrastructure
CI/CD

Automate the path from code to production. Never manually deploy again.

What is it?

Continuous Integration (CI) automatically runs tests and builds every time code is pushed. Continuous Deployment (CD) automatically ships code to production when CI passes. Together they remove the human bottleneck from shipping software.

1
Developer pushes to GitHub
A commit or pull request triggers the pipeline.
2
CI runs automatically
Tests run, TypeScript checked, lint errors flagged, Docker image built.
3
All checks pass → CD deploys
Code automatically deployed to staging or production.
4
Checks fail → developer notified
Broken code never reaches production. The pipeline is the safety net.
Real example

Vercel is CI/CD for Next.js. Push to main → Vercel builds and deploys automatically. That's a complete CI/CD pipeline. It's free for most projects.

Part 6 · Real-Time
WebSockets

HTTP asks and waits. WebSockets open a persistent channel for real-time, bidirectional communication.

The problem with HTTP for real-time

With regular HTTP, your browser asks "any new messages?" → server responds → connection closes. For chat or live dashboards you'd ask hundreds of times per second. WebSockets solve this by keeping the connection permanently open.

HTTP Polling
Ask every second for new data. Wasteful. Slow. Doesn't scale.
WebSockets
Open once. Server pushes data as it arrives. Instant. Efficient.
Chat apps (Slack, Discord)
Live dashboards
Multiplayer games
Stock price updates
Collaborative editing (Figma, Notion)
AI streaming responses
AI streaming connection

When ChatGPT streams its response word by word — that's a WebSocket (or Server-Sent Events) connection keeping the channel open while tokens arrive. Every AI chat interface works this way.

Part 6 · Real-Time
Kafka

A distributed event streaming platform. The backbone of data pipelines at companies processing millions of events per second.

What is it?

Apache Kafka is a distributed event streaming platform. Services publish events ("order placed," "user signed up") to Kafka topics. Other services subscribe to those topics and react. This decouples your architecture completely.

When you actually need it

Not for your first startup. Use it when you have multiple services reacting to the same events, or when you need high-throughput event streaming. For small apps, BullMQ (Redis-based) is usually enough.

Part 7 · Artificial Intelligence
Large Language Models

The technology behind ChatGPT, Claude, and Gemini. Understanding LLMs changes how you build software.

What is an LLM?

A Large Language Model is a neural network trained on vast amounts of text to predict the next token in a sequence. Through this training, models develop the ability to write code, answer questions, reason through problems, and generate human-like text.

The intuition

An LLM doesn't "know" things like a database does. It has learned statistical patterns in language. When you ask "What is the capital of France?" it predicts "Paris" is the most likely next token — because that pattern appeared billions of times in training data.

ConceptWhat it means
Context windowHow much text the model sees at once. GPT-4 = 128k tokens.
TemperatureRandomness. 0 = deterministic. 1+ = creative/unpredictable.
System promptInstructions given before the user message. Sets behavior.
TokensThe unit LLMs process. ~4 characters per token. You pay per token.
Fine-tuningTraining an existing model further on your specific data.

Part 7 · Artificial Intelligence
RAG & Embeddings

How to give an AI model knowledge about YOUR data. The technique behind most production AI applications.

The problem

Claude and ChatGPT don't know your company's documentation, your codebase, or last week's news. RAG (Retrieval-Augmented Generation) solves this by finding relevant information and feeding it into the model's context at query time.

What are embeddings?

An embedding converts text into a list of numbers (a vector) capturing the meaning of that text. Texts with similar meaning have vectors that are mathematically close. This enables semantic search — search by meaning, not just keywords.

1
Ingest your documents
Chunk your docs. Convert each chunk to an embedding vector. Store in a vector database.
2
User asks a question
Convert the question to a vector using the same model.
3
Find similar chunks
Search the vector DB for chunks whose vectors are closest to the question.
4
Augment the prompt
"Answer this using these documents: [chunks]" — model answers with your actual data.

Part 7 · Artificial Intelligence
AI Agents & MCP

AI that takes action. Not just answering questions — autonomously using tools, writing code, and completing tasks.

What is an AI agent?

An AI agent is an LLM with the ability to use tools — search the web, run code, call APIs, read files, take actions. Instead of "how do I fix this bug?" an agent can fix the bug, run the tests, and commit the fix.

1
Receive a goal
"Research the top 5 competitors and write a summary."
2
Plan steps
The LLM reasons which tools to use and in what order.
3
Execute with tools
Calls web search, reads pages, extracts data, repeats as needed.
4
Evaluate and return
Checks if goal is achieved. Returns completed output.

MCP — Model Context Protocol

MCP is an open standard created by Anthropic that defines a universal protocol for connecting AI models to tools and data sources. An MCP server exposes tools; any MCP-compatible AI can use them. It's the USB standard of AI tooling.

Real example

Claude Code is an AI coding agent using MCP to connect to your filesystem, terminal, and browser. It reads your codebase, runs tests, fixes bugs, and deploys — all from a conversation. This is where all software development tooling is heading.

Part 10 · Security
Security Fundamentals

Security is not a feature — it is a quality of everything you build.

HTTPS everywhere
All traffic encrypted with TLS. Vercel and Cloudflare handle this free.
Hash passwords
Never store plaintext. Use bcrypt. Never know your users' passwords.
SQL injection
Always use parameterized queries or an ORM. Never concatenate user input into SQL.
XSS prevention
Sanitize all HTML output. React does this automatically.
CORS config
Only allow requests from your actual domains. No wildcard * in production.
Rate limiting
Limit requests per IP per minute. Prevent brute force. Use Redis.
The most common breach

Most breaches aren't sophisticated — they're exposed environment variables on GitHub, SQL injection from unvalidated input, or weak admin passwords. The basics prevent 90% of attacks.
